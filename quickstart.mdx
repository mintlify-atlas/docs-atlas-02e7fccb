## Prerequisites

Before you start, make sure you have accounts and credentials for all of the following:

| Requirement | Notes |
|---|---|
| **Bun** | Runtime and package manager. [Install bun.sh](https://bun.sh) |
| **PostgreSQL** | Local install, [Neon](https://neon.tech), or [Supabase](https://supabase.com) |
| **E2B API key** | Sandbox execution platform — [e2b.dev](https://e2b.dev) |
| **Google Gemini API key** | Server-side AI — [Google AI Studio](https://aistudio.google.com/apikey) |
| **Agent Gemini API key** | Injected into sandboxes — can reuse the same key or use a separate one |
| **GitHub personal access token** | For a bot account that creates repos and PRs |
| **Upstash Redis** | Realtime pub/sub — [Upstash Console](https://console.upstash.com) |

<Tip>
  You can use the same Google Gemini API key for both `GOOGLE_GENERATIVE_AI_API_KEY` and `AGENT_GEMINI_API_KEY` during development. In production, using separate keys lets you manage quotas independently.
</Tip>

## Setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/manasseh-zw/dev0.git
    cd dev0
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    bun install
    ```
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file and fill in your credentials:

    ```bash
    cp .env.example .env.local
    ```

    Open `.env.local` and fill in all values:

    ```bash .env.local
    # Database (PostgreSQL)
    DATABASE_URL="postgresql://user:password@localhost:5432/dev0"

    # E2B Sandbox Orchestration
    E2B_API_KEY=""
    E2B_TEMPLATE="dev0-universal"
    E2B_SANDBOX_TIMEOUT_MS="3600000"

    # Google Gemini (via Vercel AI SDK)
    GOOGLE_GENERATIVE_AI_API_KEY=""

    # Agent Gemini API Key (injected into sandboxes for YOLO mode)
    AGENT_GEMINI_API_KEY=""

    # GitHub Bot Account (for repo creation & PR operations)
    GITHUB_TOKEN=""
    GITHUB_BOT_USERNAME="dev0-agent"

    # Application
    APP_URL="http://localhost:3000"
    NODE_ENV="development"

    # Upstash Realtime (Redis)
    UPSTASH_REDIS_REST_URL=""
    UPSTASH_REDIS_REST_TOKEN=""
    ```

    <Warning>
      Never commit `.env.local` to version control. It's already in `.gitignore`.
    </Warning>
  </Step>

  <Step title="Run database migrations">
    Dev0 uses Drizzle ORM with PostgreSQL. Run migrations to create the required tables:

    ```bash
    bun run db:migrate
    ```

    This creates four tables: `projects`, `tasks`, `sandboxes`, and `task_logs`, along with all enums and indexes.

    <Note>
      If you're starting fresh, you can also use `bun run db:push` to push the schema directly without migration files. Use `db:migrate` for production-like environments.
    </Note>
  </Step>

  <Step title="Start the development server">
    ```bash
    bun run dev
    ```

    The app starts on [http://localhost:3000](http://localhost:3000).
  </Step>

  <Step title="Create your first project">
    1. Open [http://localhost:3000](http://localhost:3000) in your browser
    2. Click **New project**
    3. Type a description of what you want to build (e.g., "A simple todo app with React and a REST API")
    4. Dev0's Preview Agent generates a project name, summary, and tech stack
    5. Confirm the preview to trigger the Planner Agent
    6. Gemini Pro generates a phased task graph — review the tasks
    7. Click **Start execution** to begin running tasks in E2B sandboxes
    8. Watch real-time logs stream in as each task executes
    9. When a task completes, a GitHub PR appears in the Review tab — ready for your approval
  </Step>
</Steps>

## Useful commands

| Command | Description |
|---|---|
| `bun run dev` | Start development server on port 3000 |
| `bun run build` | Build for production |
| `bun run start` | Start the production build |
| `bun run test` | Run all tests with Vitest |
| `bun run db:migrate` | Apply pending database migrations |
| `bun run db:generate` | Generate new migration files from schema changes |
| `bun run db:push` | Push schema directly to DB (development only) |
| `bun run format` | Format all files with Prettier |
