Dev0 uses a single PostgreSQL database managed with Drizzle ORM. The schema is defined in `src/lib/db/schema.ts` and consists of four tables, four enums, and a set of indexes optimized for the access patterns of the task board and execution pipeline.

## Enums

```typescript
// src/lib/db/schema.ts

export const projectStatusEnum = pgEnum('project_status', [
  'PLANNING',    // task graph is being generated
  'READY',       // all tasks created, ready to execute
  'ACTIVE',      // at least one task is running
  'PAUSED',      // execution paused by user
  'COMPLETED',   // all tasks done or skipped
  'ARCHIVED',    // hidden from default dashboard view
])

export const taskStatusEnum = pgEnum('task_status', [
  'PENDING',   // waiting to be executed
  'RUNNING',   // claimed and executing
  'REVIEW',    // PR created, awaiting human review
  'DONE',      // PR merged
  'FAILED',    // execution failed
  'SKIPPED',   // manually skipped
])

export const geminiModelEnum = pgEnum('gemini_model', [
  'gemini-3-flash-preview',  // fast, lower cost
  'gemini-3-pro-preview',    // deep reasoning, higher cost
])

export const sandboxStatusEnum = pgEnum('sandbox_status', [
  'READY',    // sandbox is running and available
  'RUNNING',  // sandbox is actively executing a task
  'STOPPED',  // sandbox has been paused or terminated
])
```

## Tables

### projects

Stores one record per project created by a user.

```typescript
export const projects = pgTable(
  'projects',
  {
    id:          uuid('id').defaultRandom().primaryKey(),
    name:        text('name').notNull(),
    description: text('description'),
    status:      projectStatusEnum('status').default('PLANNING').notNull(),
    repoUrl:     text('repo_url'),       // full HTTPS URL to the GitHub repo
    repoName:    text('repo_name'),      // owner/repo slug (e.g. 'myorg/myapp')
    techStack:   text('tech_stack').default('react-vite').notNull(),
    theme:       text('theme').default('slate').notNull(),
    vibeInput:   text('vibe_input'),     // original user idea text
    specContent: text('spec_content'),   // Planner Agent-generated spec (full text)
    createdAt:   timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
    updatedAt:   timestamp('updated_at', { withTimezone: true }).defaultNow().$onUpdate(() => new Date()).notNull(),
  },
  (table) => [index('projects_status_idx').on(table.status)],
)
```

<ResponseField name="id" type="uuid" required>
  Auto-generated UUID primary key.
</ResponseField>

<ResponseField name="name" type="text" required>
  Human-readable project name generated by the Preview Agent.
</ResponseField>

<ResponseField name="status" type="projectStatusEnum" required>
  Current lifecycle state. Defaults to `PLANNING`.
</ResponseField>

<ResponseField name="repoUrl" type="text">
  Full HTTPS clone URL for the GitHub repository. Set after the planner creates the repo.
</ResponseField>

<ResponseField name="repoName" type="text">
  Short `owner/name` slug. Used to construct authenticated clone URLs when `repoUrl` is present but the token needs embedding.
</ResponseField>

<ResponseField name="vibeInput" type="text">
  The original free-text idea the user typed. Stored for audit and replay purposes.
</ResponseField>

<ResponseField name="specContent" type="text">
  The full project specification generated by the Planner Agent. Included in every task prompt so Gemini CLI has full project context.
</ResponseField>

### tasks

Stores one record per planned task. Tasks are the atomic unit of work in Dev0.

```typescript
export const tasks = pgTable(
  'tasks',
  {
    id:           uuid('id').defaultRandom().primaryKey(),
    projectId:    uuid('project_id').notNull().references(() => projects.id, { onDelete: 'cascade' }),
    title:        text('title').notNull(),
    description:  text('description'),
    phase:        integer('phase').default(1).notNull(),
    order:        integer('order').default(0).notNull(),
    status:       taskStatusEnum('status').default('PENDING').notNull(),
    geminiModel:  geminiModelEnum('gemini_model').default('gemini-3-pro-preview').notNull(),
    dependencies: text('dependencies').array().notNull().default([]),
    prUrl:        text('pr_url'),
    prNumber:     integer('pr_number'),
    logs:         jsonb('logs').$type<{}[] | null>(),
    attempts:     integer('attempts').default(0).notNull(),
    maxAttempts:  integer('max_attempts').default(3).notNull(),
    createdAt:    timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
    updatedAt:    timestamp('updated_at', { withTimezone: true }).defaultNow().$onUpdate(() => new Date()).notNull(),
  },
  (table) => [
    index('tasks_project_idx').on(table.projectId),
    index('tasks_status_idx').on(table.status),
    index('tasks_project_phase_order_idx').on(table.projectId, table.phase, table.order),
  ],
)
```

<ResponseField name="phase" type="integer" required>
  Phase number (1–5). Tasks in higher phases do not start until all tasks in lower phases are `DONE` or `SKIPPED`.
</ResponseField>

<ResponseField name="order" type="integer" required>
  Position within the phase (1-based). Used for display ordering and the `getNextRunnableTask()` sort.
</ResponseField>

<ResponseField name="geminiModel" type="geminiModelEnum" required>
  The Gemini model to use for this task's execution. Assigned by the Planner Agent based on task complexity.
</ResponseField>

<ResponseField name="dependencies" type="text[]" required>
  Array of task UUIDs that must be in `DONE` or `SKIPPED` status before this task can be executed. Stored as a PostgreSQL text array.
</ResponseField>

<ResponseField name="prUrl" type="text">
  GitHub PR URL. Set when the task transitions to `REVIEW`.
</ResponseField>

<ResponseField name="prNumber" type="integer">
  GitHub PR number extracted from `prUrl`. Used to fetch PR details and diffs via the Octokit API.
</ResponseField>

<ResponseField name="attempts" type="integer" required>
  Number of times this task has been attempted. Incremented on every `FAILED` transition. Defaults to 0.
</ResponseField>

<ResponseField name="maxAttempts" type="integer" required>
  Maximum number of execution attempts before the task is considered permanently failed. Defaults to 3.
</ResponseField>

### sandboxes

Tracks E2B sandbox instances. One sandbox can be reused across multiple tasks within the same project.

```typescript
export const sandboxes = pgTable(
  'sandboxes',
  {
    id:         uuid('id').defaultRandom().primaryKey(),
    projectId:  uuid('project_id').notNull().references(() => projects.id, { onDelete: 'cascade' }),
    sandboxId:  text('sandbox_id').notNull().unique(),  // E2B sandbox ID
    status:     sandboxStatusEnum('status').default('READY').notNull(),
    taskId:     uuid('task_id').unique(),               // currently assigned task
    snapshotId: text('snapshot_id'),                   // E2B template name (e.g. 'dev0-universal')
    publicUrl:  text('public_url'),                    // exposed port URL (for dev server)
    createdAt:  timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
    updatedAt:  timestamp('updated_at', { withTimezone: true }).defaultNow().$onUpdate(() => new Date()).notNull(),
  },
  (table) => [
    index('sandboxes_project_idx').on(table.projectId),
    index('sandboxes_status_idx').on(table.status),
  ],
)
```

<ResponseField name="sandboxId" type="text" required>
  The E2B-assigned sandbox identifier (different from the DB `id`). Used to connect via `Sandbox.connect(sandboxId)`.
</ResponseField>

<ResponseField name="taskId" type="uuid">
  The task currently using this sandbox. Unique constraint ensures one sandbox per task at most. Nullable — a sandbox can exist without a task assignment.
</ResponseField>

<ResponseField name="snapshotId" type="text">
  The E2B template name used to create this sandbox (e.g., `dev0-universal`). Useful for debugging and auditing which template version was used.
</ResponseField>

<ResponseField name="publicUrl" type="text">
  Public HTTPS URL for an exposed sandbox port, used when a dev server is running inside the sandbox for preview.
</ResponseField>

### task_logs

Stores the full Gemini CLI event log for each task execution. Separated from `tasks` to enable lazy loading — the events array can be large and is only needed when viewing task details.

```typescript
export const taskLogs = pgTable(
  'task_logs',
  {
    id:             uuid('id').defaultRandom().primaryKey(),
    taskId:         uuid('task_id').notNull().references(() => tasks.id, { onDelete: 'cascade' }).unique(),
    events:         jsonb('events').$type<GeminiStreamEvent[]>().notNull().default([]),
    summary:        text('summary'),            // last assistant message (max 2000 chars)
    totalTokens:    integer('total_tokens'),    // from GeminiResultEvent.stats
    durationMs:     integer('duration_ms'),     // ms from start to finish
    toolCallsCount: integer('tool_calls_count').default(0),
    createdAt:      timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
    updatedAt:      timestamp('updated_at', { withTimezone: true }).defaultNow().$onUpdate(() => new Date()).notNull(),
  },
  (table) => [index('task_logs_task_id_idx').on(table.taskId)],
)
```

<ResponseField name="events" type="jsonb (GeminiStreamEvent[])" required>
  Full array of all Gemini CLI events from the execution session, stored as JSONB. Default is an empty array.
</ResponseField>

<ResponseField name="summary" type="text">
  The last non-delta assistant message from the session. Truncated to 2000 characters. Displayed as a quick summary in the task detail view without loading the full events array.
</ResponseField>

<ResponseField name="totalTokens" type="integer">
  Total tokens consumed by the session (input + output), from `GeminiResultEvent.stats.total_tokens`.
</ResponseField>

<ResponseField name="durationMs" type="integer">
  Wall-clock duration of the Gemini CLI process in milliseconds. Falls back to the orchestrator's measured duration if the `result` event does not include `duration_ms`.
</ResponseField>

<ResponseField name="toolCallsCount" type="integer">
  Count of `tool_use` events in the session. Indicates how many tool invocations Gemini made.
</ResponseField>

## Relations

```typescript
// projects → tasks (one-to-many)
export const projectRelations = relations(projects, ({ many }) => ({
  tasks: many(tasks),
  sandboxes: many(sandboxes),
}))

// tasks → project, sandbox, executionLogs (many-to-one)
export const taskRelations = relations(tasks, ({ one }) => ({
  project: one(projects, {
    fields: [tasks.projectId],
    references: [projects.id],
  }),
  sandbox: one(sandboxes, {
    fields: [tasks.id],
    references: [sandboxes.taskId],
  }),
  executionLogs: one(taskLogs, {
    fields: [tasks.id],
    references: [taskLogs.taskId],
  }),
}))

// sandboxes → project, task (many-to-one)
export const sandboxRelations = relations(sandboxes, ({ one }) => ({
  project: one(projects, {
    fields: [sandboxes.projectId],
    references: [projects.id],
  }),
  task: one(tasks, {
    fields: [sandboxes.taskId],
    references: [tasks.id],
  }),
}))

// task_logs → task (many-to-one)
export const taskLogsRelations = relations(taskLogs, ({ one }) => ({
  task: one(tasks, {
    fields: [taskLogs.taskId],
    references: [tasks.id],
  }),
}))
```

## Indexes

| Index | Table | Columns | Purpose |
|-------|-------|---------|--------|
| `projects_status_idx` | `projects` | `status` | Filter projects by status on the dashboard |
| `tasks_project_idx` | `tasks` | `project_id` | Fetch all tasks for a project |
| `tasks_status_idx` | `tasks` | `status` | Filter tasks by status (e.g., all `RUNNING`) |
| `tasks_project_phase_order_idx` | `tasks` | `project_id, phase, order` | Ordered task queries in `getNextRunnableTask()` — the most frequent orchestrator query |
| `sandboxes_project_idx` | `sandboxes` | `project_id` | Find sandboxes for a project in `getOrCreateProjectSandbox()` |
| `sandboxes_status_idx` | `sandboxes` | `status` | Filter for `READY` sandboxes |
| `task_logs_task_id_idx` | `task_logs` | `task_id` | Fast lookup by task ID when loading execution logs |

<Note>
  The `tasks_project_phase_order_idx` composite index is the most critical for performance. `getNextRunnableTask()` runs on every `startTask()` call and scans all tasks for a project ordered by phase and order. Without this index, task board performance degrades linearly with task count.
</Note>

## Cascade deletes

Both `tasks` and `sandboxes` reference `projects.id` with `{ onDelete: 'cascade' }`. Deleting a project removes all its tasks, sandboxes, and (via `task_logs` cascade from `tasks`) all execution logs.

`task_logs` references `tasks.id` with `{ onDelete: 'cascade' }`, so deleting a task also removes its log record.

## Database migrations

Drizzle Kit manages schema migrations:

```bash
# Generate a new migration from schema changes
bunx drizzle-kit generate

# Apply pending migrations to the database
bunx drizzle-kit migrate

# Push schema directly (no migration files — for development only)
bunx drizzle-kit push
```

The `db:migrate` npm script wraps `drizzle-kit migrate` and is the standard way to apply migrations in CI and production deployments.
